# 画面仕様書 — Pokemon TCGP Win Track

---

## 1. 画面一覧

本アプリケーションは**単一画面アプリケーション**であり、画面遷移は存在しない。
同一画面内で「通常モード」と「配信モード」の2つの表示モードを切り替える。

| モード | 説明 |
|--------|------|
| 通常モード | 全UI要素を表示する。デフォルトの表示状態 |
| 配信モード | ポイント・連勝数を拡大表示し、不要な要素を非表示にする |

モード切替はトグルボタンで行い、画面遷移（URLやルーティングの変更）は発生しない。

---

## 2. 通常モード画面仕様

### 2.1 画面の目的

ランクマッチの勝敗を記録し、ポイント・連勝数・履歴・推移グラフを一覧で確認する。

### 2.2 主な利用シーン

- 試合終了直後に勝敗ボタンをタップして記録する
- セッション中にポイント推移を確認する
- 誤入力をUndoで取り消す

### 2.3 レイアウト構成（上から順）

| 順序 | エリア名 | 含まれる要素 |
|------|---------|------------|
| 1 | ヘッダー | アプリ名、配信モード切替ボタン |
| 2 | ステータス表示 | 現在ポイント、現在連勝数 |
| 3 | 入力エリア | 勝ちボタン、負けボタン、引き分けボタン、Undoボタン |
| 4 | グラフエリア | ポイント推移折れ線グラフ |
| 5 | 履歴エリア | 試合履歴一覧（新しい順） |

### 2.4 表示項目一覧

| 項目名 | 表示内容 | 表示形式 |
|--------|---------|---------|
| アプリ名 | "Pokemon TCGP Win Track" | テキスト |
| 現在ポイント | 現在の累積ポイント | "{数値} pt"（例: "120 pt"） |
| 現在連勝数 | 現在の連続勝利数 | "{数値} 連勝"（例: "3 連勝"）。0の場合は "0 連勝" |
| ポイント推移グラフ | 試合ごとのポイント推移 | 折れ線グラフ（X軸: 試合番号、Y軸: ポイント） |
| 履歴一覧 | 全試合結果の一覧 | 新しい順で表示。1行あたり: 試合結果、ポイント変動値、記録後ポイント、記録日時 |

### 2.5 ボタン一覧

| ボタン名 | ラベル | 役割 |
|---------|-------|------|
| 勝ちボタン | "Win" | 勝利を記録する |
| 負けボタン | "Lose" | 敗北を記録する |
| 引き分けボタン | "Draw" | 引き分けを記録する |
| Undoボタン | "Undo" | 直前の記録を取り消す |
| 配信モード切替ボタン | （トグル形式） | 通常モード/配信モードを切り替える |

## 3. 配信モード画面仕様

### 3.1 画面の目的

配信画面上でポイントと連勝数を視聴者に分かりやすく表示する。操作UIは維持し、不要な情報を非表示にする。

### 3.2 通常モードとの差分

| 要素 | 通常モード | 配信モード | 差分 |
|------|-----------|-----------|------|
| 現在ポイント | 通常サイズで表示 | 拡大表示（通常時の2倍以上のフォントサイズ） | サイズ変更 |
| 現在連勝数 | 通常サイズで表示 | 拡大表示（通常時の2倍以上のフォントサイズ） | サイズ変更 |
| 勝ちボタン | 表示 | 表示（サイズは通常時と同一） | 変更なし |
| 負けボタン | 表示 | 表示（サイズは通常時と同一） | 変更なし |
| 引き分けボタン | 表示 | 表示（サイズは通常時と同一） | 変更なし |
| Undoボタン | 表示 | **非表示** | 非表示化 |
| ポイント推移グラフ | 表示 | **非表示** | 非表示化 |
| 履歴一覧 | 表示 | **非表示** | 非表示化 |
| 配信モード切替ボタン | 表示 | 表示 | 変更なし（モード解除のため常時表示） |

### 3.3 フォントサイズ・視認性方針

- 現在ポイントと連勝数は通常時の2倍以上のフォントサイズで表示する
- 1080p配信において視聴者が判読可能なサイズとする（NF-5準拠）
- 勝敗入力ボタンのサイズは通常時と同一とし、操作性を維持する

### 3.4 配信モードで残す情報

- 現在ポイント（拡大）
- 現在連勝数（拡大）
- 勝ち / 負け / 引き分けボタン
- 配信モード切替ボタン

### 3.5 配信モードで隠す情報

- Undoボタン
- ポイント推移グラフ
- 履歴一覧

## 4. UI要素詳細

### 4.1 要素定義一覧

| 要素名 | 種別 | 説明 | 入力可否 | 活性条件 | 備考 |
|--------|------|------|---------|---------|------|
| アプリ名 | 表示 | アプリケーション名称を表示 | 不可 | 常に表示 | — |
| 配信モード切替ボタン | 切替（トグル） | 通常モード/配信モードを切り替える | 可 | 常に活性 | 両モードで常時表示 |
| 現在ポイント | 表示 | 現在の累積ポイントを数値表示 | 不可 | 常に表示 | 配信モード時は拡大表示 |
| 現在連勝数 | 表示 | 現在の連続勝利数を数値表示 | 不可 | 常に表示 | 配信モード時は拡大表示 |
| 勝ちボタン | ボタン | 勝利を記録する | 可 | 常に活性 | ラベル: "Win" |
| 負けボタン | ボタン | 敗北を記録する | 可 | 常に活性 | ラベル: "Lose" |
| 引き分けボタン | ボタン | 引き分けを記録する | 可 | 常に活性 | ラベル: "Draw" |
| Undoボタン | ボタン | 直前の記録を取り消す | 可 | canUndoがtrueの場合のみ活性 | canUndoがfalseの場合はdisabled。配信モード時は非表示 |
| ポイント推移グラフ | グラフ | ポイント推移を折れ線グラフで表示 | 不可 | 常に表示（通常モード時） | 配信モード時は非表示 |
| 履歴一覧 | 表示（リスト） | 全試合履歴を新しい順で表示 | 不可 | 常に表示（通常モード時） | 配信モード時は非表示 |

### 4.2 Undoボタンの活性 / 非活性条件

| 条件 | canUndoの値 | ボタン状態 |
|------|-----------|-----------|
| アプリ起動直後（履歴0件） | false | 非活性（disabled） |
| アプリ起動直後（LocalStorageから復元後） | false | 非活性（disabled） |
| 試合記録入力後 | true | 活性 |
| Undo実行後 | false | 非活性（disabled） |
| 連続Undo試行時 | false | 非活性（disabled）のため操作不可 |

### 4.3 表示 / 非表示条件まとめ

| 要素 | 通常モード | 配信モード |
|------|-----------|-----------|
| アプリ名 | 表示 | 表示 |
| 配信モード切替ボタン | 表示 | 表示 |
| 現在ポイント | 表示 | 表示（拡大） |
| 現在連勝数 | 表示 | 表示（拡大） |
| 勝ちボタン | 表示 | 表示 |
| 負けボタン | 表示 | 表示 |
| 引き分けボタン | 表示 | 表示 |
| Undoボタン | 表示（活性/非活性あり） | 非表示 |
| ポイント推移グラフ | 表示 | 非表示 |
| 履歴一覧 | 表示 | 非表示 |

## 5. 状態変化と画面反映

### 5.1 勝ち入力時

| 更新対象 | 変化内容 |
|---------|---------|
| 現在ポイント | 増加値（10 + min(記録前連勝数, 4)）を加算した値に更新 |
| 現在連勝数 | +1 した値に更新 |
| Undoボタン | 活性化（canUndo = true） |
| ポイント推移グラフ | 新しいプロットポイントが追加される |
| 履歴一覧 | 最上部に新しい試合記録が追加される。表示: "Win", "+{増加値}", "→ {記録後ポイント} pt", "{時刻}" |

### 5.2 負け入力時

| 更新対象 | 変化内容 |
|---------|---------|
| 現在ポイント | 5を減算した値に更新（下限0） |
| 現在連勝数 | 0 にリセット |
| Undoボタン | 活性化（canUndo = true） |
| ポイント推移グラフ | 新しいプロットポイントが追加される |
| 履歴一覧 | 最上部に新しい試合記録が追加される。表示: "Lose", "-5", "→ {記録後ポイント} pt", "{時刻}" |

### 5.3 引き分け入力時

| 更新対象 | 変化内容 |
|---------|---------|
| 現在ポイント | 変化なし |
| 現在連勝数 | 0 にリセット |
| Undoボタン | 活性化（canUndo = true） |
| ポイント推移グラフ | 新しいプロットポイントが追加される（ポイント値は前回と同一） |
| 履歴一覧 | 最上部に新しい試合記録が追加される。表示: "Draw", "0", "→ {記録後ポイント} pt", "{時刻}" |

### 5.4 Undo時

| 更新対象 | 変化内容 |
|---------|---------|
| 現在ポイント | 取り消した試合記録のpointBeforeに復元 |
| 現在連勝数 | 取り消した試合記録のwinStreakBeforeに復元 |
| Undoボタン | 非活性化（canUndo = false） |
| ポイント推移グラフ | 末尾のプロットポイントが削除される |
| 履歴一覧 | 最上部の試合記録が削除される |

### 5.5 配信モード切替時

| 操作 | 変化内容 |
|------|---------|
| 通常 → 配信モード | 現在ポイント・連勝数が拡大表示。Undoボタン・グラフ・履歴が非表示。ポイント・連勝数の値自体は変化しない |
| 配信 → 通常モード | 全UI要素が通常サイズで表示。非表示だった要素が再表示される。状態値は変化しない |

## 6. 履歴一覧の表示仕様

### 6.1 1行あたりの表示項目

| 表示項目 | 表示形式 | 例 |
|---------|---------|-----|
| 試合結果 | "Win" / "Lose" / "Draw" | "Win" |
| ポイント変動値 | "+{値}" / "-{値}" / "0" | "+12" |
| 記録後ポイント | "→ {値} pt" | "→ 120 pt" |
| 記録日時 | "HH:MM" | "14:30" |

### 6.2 表示順序

新しい試合が最上部、古い試合が最下部（時系列降順）。

### 6.3 試合記録が0件の場合

履歴エリアは空状態で表示する。

---

## 7. グラフ表示仕様

### 7.1 グラフ種別

折れ線グラフ。

### 7.2 軸

| 軸 | 内容 | 単位 |
|----|------|------|
| X軸 | 試合番号 | 1始まりの連番（0は初期状態） |
| Y軸 | ポイント | 整数値 |

### 7.3 プロットポイント

- 原点: 試合0回目、ポイント0（常にプロット）
- 各試合後のpointAfterを順にプロット
- 試合3件の場合、プロットは4点

### 7.4 試合記録が0件の場合

原点（試合0回目、ポイント0）のみ表示する。

### 7.5 更新タイミング

試合記録入力の直後、およびUndo実行の直後に更新する。

---

## 8. モバイル表示方針

### 8.1 対応画面幅

横幅320px以上で全機能が操作可能であること（NF-4準拠）。

### 8.2 縦画面時の並び順

通常モードのレイアウト構成（2.3節）と同一の上下配置とする。
横並びレイアウトへの切り替えは行わない。

| 順序 | エリア | 備考 |
|------|--------|------|
| 1 | ヘッダー | — |
| 2 | ステータス表示 | — |
| 3 | 入力エリア | ボタンは横並び配置。画面幅に応じて折り返し可 |
| 4 | グラフエリア | 画面幅に合わせてグラフ幅を伸縮 |
| 5 | 履歴エリア | — |

### 8.3 小画面時の優先表示

| 優先度 | 要素 | 理由 |
|--------|------|------|
| 必須 | 現在ポイント | 最も参照頻度が高い情報 |
| 必須 | 勝ち / 負け / 引き分けボタン | 主操作 |
| 必須 | 現在連勝数 | ポイント計算に影響する情報 |
| 高 | Undoボタン | 誤操作対応 |
| 中 | グラフエリア | スクロールで到達可能 |
| 中 | 履歴エリア | スクロールで到達可能 |

### 8.4 折りたたみ・省略

MVPでは折りたたみ・省略は行わない。全要素を縦スクロールで表示する。

---

## 9. 画面遷移の有無

本アプリケーションは**単一画面アプリケーション**である。
ルーティングやページ遷移は存在しない。
通常モードと配信モードの切り替えは、同一画面内での表示モード変更として実現する。

---

## 10. エラー表示仕様

### 10.1 エラー通知方式

トースト通知（画面の操作を妨げない形式）で表示する。

### 10.2 エラーメッセージ

| 状況 | メッセージ |
|------|----------|
| LocalStorage読み込み失敗 | 「前回のデータを読み込めませんでした。新しく開始します。」 |
| LocalStorage書き込み失敗 | 「データの保存に失敗しました。ブラウザの空き容量を確認してください。」 |

技術的なエラーメッセージ（例: "JSON parse error"）はユーザーに表示しない。
