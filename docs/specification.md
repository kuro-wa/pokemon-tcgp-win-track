# 仕様書 — Pokemon TCGP Win Track

---

## 1. 状態定義

アプリケーションが管理する状態は以下の通り。

| 状態名 | 型 | 初期値 | 説明 |
|--------|-----|--------|------|
| point | 整数 | 0 | 現在の累積ポイント |
| winStreak | 整数（0以上） | 0 | 現在の連続勝利数 |
| matchHistory | 試合記録の配列 | 空配列 | 全試合結果の時系列リスト（古い順） |
| isStreamMode | 真偽値 | false | 配信モードの有効/無効 |

### 1.1 試合記録の構造

1件の試合記録は以下の要素を持つ。

| 要素名 | 型 | 説明 |
|--------|-----|------|
| result | "win" / "lose" / "draw" | 試合結果 |
| pointBefore | 整数 | この試合の直前のポイント |
| pointAfter | 整数 | この試合の直後のポイント |
| winStreakBefore | 整数（0以上） | この試合の直前の連勝数 |
| winStreakAfter | 整数（0以上） | この試合の直後の連勝数 |
| timestamp | ISO 8601文字列 | 記録日時（例: "2026-04-05T14:30:00.000Z"） |

## 2. 勝敗入力仕様

### 2.1 勝ち（Win）

- ポイントにポイント増加値（後述のポイント計算仕様で算出）を加算する
- 連勝数を +1 する

### 2.2 負け（Lose）

- ポイントからポイント減少値（後述のポイント計算仕様で算出）を減算する
- 連勝数を 0 にリセットする

### 2.3 引き分け（Draw）

- ポイントは変動しない（増減値は 0）
- 連勝数を 0 にリセットする

### 2.4 共通処理

勝敗入力時、以下の処理を順序通り実行する。

1. 現在の point と winStreak を記録する（pointBefore, winStreakBefore）
2. 試合結果に応じて point と winStreak を更新する
3. 更新後の point と winStreak を記録する（pointAfter, winStreakAfter）
4. 試合記録を matchHistory の末尾に追加する
5. 全状態を LocalStorage に保存する

---

## 3. ポイント計算仕様

### 3.1 基本増減値

| 試合結果 | 基本増減値 |
|---------|-----------|
| 勝ち | +10 |
| 負け | -5 |
| 引き分け | 0 |

### 3.2 連勝ボーナス

勝利時、**その勝利を記録する前の連勝数**に基づき、基本増加値にボーナスを加算する。

| 記録前の連勝数 | ボーナス値 | 合計増加値 |
|--------------|-----------|-----------|
| 0（初勝利） | 0 | +10 |
| 1 | +1 | +11 |
| 2 | +2 | +12 |
| 3 | +3 | +13 |
| 4以上 | +4 | +14 |

連勝ボーナスの上限は +4 とする。連勝数が4以上の場合、ボーナスは一律 +4 で固定される。

### 3.3 ポイント増加値の計算式

```
勝利時のポイント増加値 = 基本増加値(10) + min(記録前の連勝数, 4)
```

### 3.4 ポイント減少値

負け時のポイント減少値は常に 5 とする。連勝数やポイント残高による変動はない。

### 3.5 ポイント下限

ポイントの下限は 0 とする。減算の結果が負の値になる場合、ポイントは 0 に切り上げる。

### 3.6 ポイント上限

ポイントに上限は設けない。

### 3.7 計算例

| 操作順 | 試合結果 | 記録前連勝数 | ポイント増減 | 記録後ポイント | 記録後連勝数 |
|--------|---------|------------|------------|--------------|------------|
| 1 | 勝ち | 0 | +10 | 10 | 1 |
| 2 | 勝ち | 1 | +11 | 21 | 2 |
| 3 | 勝ち | 2 | +12 | 33 | 3 |
| 4 | 負け | 3 | -5 | 28 | 0 |
| 5 | 引き分け | 0 | 0 | 28 | 0 |
| 6 | 勝ち | 0 | +10 | 38 | 1 |

## 4. Undo仕様

### 4.1 取り消し対象

直前に記録した1件の試合のみ取り消し可能とする。

### 4.2 取り消し時の復元処理

1. matchHistory の末尾から1件を取り除く
2. 取り除いた試合記録の pointBefore を現在の point に復元する
3. 取り除いた試合記録の winStreakBefore を現在の winStreak に復元する
4. 全状態を LocalStorage に保存する

### 4.3 連続Undo

連続Undoは不可とする。
Undo実行後、次の試合を記録するまでUndoボタンは無効化される。

### 4.4 Undo不可条件

以下の場合、Undoは実行できない。

- matchHistory が空（試合記録が0件）
- 直前の操作がUndo（連続Undo禁止）

### 4.5 Undo可否の判定状態

Undo可否を判定するため、以下の状態を管理する。

| 状態名 | 型 | 初期値 | 説明 |
|--------|-----|--------|------|
| canUndo | 真偽値 | false | Undoボタンの有効/無効 |

canUndo の遷移ルール:

- 試合記録時: true に設定する
- Undo実行時: false に設定する
- アプリ起動時（LocalStorageから復元時）: false に設定する

## 5. 履歴仕様

### 5.1 記録項目

1件の試合記録が保持する項目は「1.1 試合記録の構造」に定義した通り。

### 5.2 保存順序

matchHistory 配列は時系列昇順（古い順）で格納する。
新しい記録は配列の末尾に追加する。

### 5.3 表示順序

UI上での履歴表示は時系列降順（新しい順）とする。
最新の試合結果が最上部に表示される。

### 5.4 保存件数上限

履歴の保存件数に上限は設けない。

---

## 6. グラフ仕様

### 6.1 グラフ種別

折れ線グラフを使用する。

### 6.2 軸定義

| 軸 | 内容 | 単位 |
|----|------|------|
| X軸 | 試合番号 | 1始まりの連番（1, 2, 3, ...） |
| Y軸 | ポイント | 整数値 |

### 6.3 プロットポイント

- 初期状態（試合0回目、ポイント0）を原点としてプロットする
- 各試合後のポイント（pointAfter）を順にプロットする
- 例: 試合が3件の場合、プロットは4点（初期値 + 3試合分）

### 6.4 更新タイミング

- 試合記録入力の直後にグラフを更新する
- Undo実行の直後にグラフを更新する

### 6.5 試合記録が0件の場合

原点（試合0回目、ポイント0）のみを表示する。

## 7. 永続化仕様

### 7.1 保存対象

以下の状態を1つのオブジェクトとしてLocalStorageに保存する。

| 保存対象 | 説明 |
|---------|------|
| point | 現在ポイント |
| winStreak | 現在連勝数 |
| matchHistory | 全試合記録 |

canUndo と isStreamMode は保存しない。起動時にそれぞれ false で初期化する。

### 7.2 保存タイミング

以下のイベント発生時に保存を実行する。

- 試合結果の記録時
- Undo実行時

### 7.3 保存キー

LocalStorage のキーは `"pokemon-tcgp-win-track"` とする。

### 7.4 保存形式

JSON文字列として保存する。

### 7.5 読み込みタイミング

アプリケーション起動時（初回レンダリング前）に1回読み込む。

### 7.6 読み込み失敗時のフォールバック

以下のいずれかに該当する場合、全状態を初期値で起動する。

| 失敗パターン | 挙動 |
|-------------|------|
| キーが存在しない | 初期値で起動 |
| 値がnullまたはundefined | 初期値で起動 |
| JSON.parseが失敗する | 初期値で起動 |
| パース結果が期待する構造と一致しない | 初期値で起動 |

初期値:

- point: 0
- winStreak: 0
- matchHistory: 空配列

フォールバック時、破損データは上書き保存しない。次回の状態変更時に正常データで上書きされる。

## 8. 配信モード仕様

### 8.1 切り替え方法

配信モードのON/OFFをトグルボタンで切り替える。

### 8.2 配信モードON時の表示

| 要素 | 表示 | 備考 |
|------|------|------|
| 現在ポイント | 表示（拡大） | 通常時の2倍以上のフォントサイズ |
| 連勝数 | 表示（拡大） | 通常時の2倍以上のフォントサイズ |
| 勝敗入力ボタン | 表示 | サイズは通常時と同一 |
| グラフ | 非表示 | — |
| 履歴一覧 | 非表示 | — |
| Undoボタン | 非表示 | — |
| 配信モード切替ボタン | 表示 | モード解除のため常時表示 |

### 8.3 配信モードOFF時の表示

全UI要素を通常サイズで表示する（デフォルト状態）。

### 8.4 永続化

配信モードの状態はLocalStorageに保存しない。アプリ起動時は常にOFF（通常モード）で開始する。

## 9. エラーハンドリング仕様

### 9.1 LocalStorage関連エラー

| エラー種別 | 発生タイミング | 対応 |
|-----------|-------------|------|
| 読み込み時のJSON解析失敗 | アプリ起動時 | 初期状態で起動する。アプリは正常に利用可能 |
| 読み込み時のデータ構造不正 | アプリ起動時 | 初期状態で起動する。アプリは正常に利用可能 |
| 書き込み失敗（容量超過等） | 状態変更時 | アプリの動作は継続する。メモリ上の状態は正常に保持される。ユーザーに保存失敗を通知する |

### 9.2 UI上のエラー表示方針

- 技術的なエラーメッセージ（例: "JSON parse error"）をユーザーに表示しない
- ユーザーが理解できる平易な言葉で通知する
- エラー通知はアプリの操作を妨げない形式（トースト通知等）で表示する

### 9.3 エラー通知メッセージ

| 状況 | メッセージ |
|------|----------|
| LocalStorage読み込み失敗 | 「前回のデータを読み込めませんでした。新しく開始します。」 |
| LocalStorage書き込み失敗 | 「データの保存に失敗しました。ブラウザの空き容量を確認してください。」 |

## 10. 画面構成仕様

### 10.1 通常モードのレイアウト（上から順）

1. **ヘッダー**: アプリ名、配信モード切替ボタン
2. **ステータス表示**: 現在ポイント、現在連勝数
3. **入力エリア**: 勝ちボタン、負けボタン、引き分けボタン、Undoボタン
4. **グラフエリア**: ポイント推移グラフ
5. **履歴エリア**: 試合履歴一覧（新しい順）

### 10.2 各要素の表示内容

#### ステータス表示

- 現在ポイント: 数値を表示する（例: "120 pt"）
- 現在連勝数: 数値を表示する（例: "3 連勝"）。連勝数が0の場合は "0 連勝" と表示する

#### 入力ボタン

- 勝ちボタン: ラベル「Win」
- 負けボタン: ラベル「Lose」
- 引き分けボタン: ラベル「Draw」
- Undoボタン: ラベル「Undo」。canUndoがfalseの場合は無効化（disabled）表示する

#### 履歴一覧の1行あたりの表示

- 試合結果（Win / Lose / Draw）
- ポイント変動値（例: "+10", "-5", "0"）
- 記録後ポイント（例: "→ 120 pt"）
- 記録日時（例: "14:30"）
