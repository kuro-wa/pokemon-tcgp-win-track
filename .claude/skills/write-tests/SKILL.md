# Test Writing Skill

## トリガー条件

以下の依頼時にこの skill を適用する:
- 新機能の実装前のテスト作成
- 既存機能の変更前のテスト更新
- テストカバレッジの追加
- バグ修正時の再現テスト作成

## TDD 前提条件

- この skill は他の write-* skill より先に実行すること。
- 実装対象の仕様・要件が明確であることを確認してからテストを作成すること。

## 実装ルール

- Given / When / Then 形式で記述すること。
- 正常系・異常系・境界値を網羅すること。
- テストは可読性を最優先とすること。
- テストファイルは `tests/` 配下に、src のディレクトリ構造をミラーして配置すること。
- テストファイル名は `{対象モジュール名}.test.ts` または `.test.tsx` とすること。
