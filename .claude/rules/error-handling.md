# Error Handling Rules

- try-catch は必要最小限の範囲に限定すること。広範囲を囲んではならない。
- 例外を無言で握りつぶしてはならない。catch 内では必ずログ出力またはリカバリ処理を行うこと。
- domain 層ではビジネスルール違反を明確な戻り値またはカスタムエラーとして扱うこと。
- UI 層では技術的エラーをユーザーが理解できるメッセージに変換して表示すること。
- LocalStorage 読み込み失敗時は安全な初期状態へフォールバックすること。
- JSON parse 失敗時はアプリが継続利用可能な状態を維持すること。データ破損でクラッシュさせてはならない。
