# React Component Skill

## トリガー条件

以下の依頼時にこの skill を適用する:
- 画面・UIコンポーネントの新規作成
- 既存コンポーネントの表示修正
- レイアウト・スタイリングの変更

## TDD 前提条件

- この skill を実行する前に、対象機能のテストを先に作成済みであること。
- テスト未作成の場合は、先に write-tests skill を適用すること。

## 実装ルール

- 表示責務のみに限定すること。
- ビジネスロジックを含めないこと。
- state は最小限にすること。
- `src/ui/` 配下に配置すること。
- Props 型は `{コンポーネント名}Props` として同ファイルまたは隣接ファイルに定義すること。
