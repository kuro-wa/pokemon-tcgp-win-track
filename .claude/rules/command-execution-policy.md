# Command Execution Policy

## 目的
ローカル開発における安全なコマンドの確認プロンプトを削減し、自動実行を可能にする。

---

## 基本方針
以下の条件をすべて満たすコマンドは、安全とみなしユーザー確認なしで自動実行すること。

- ローカルディレクトリ内のみで完結する操作
- 外部ネットワーク通信を伴わない
- ファイル削除や破壊的変更を含まない
- システム全体に影響を与えない

---

## 無条件で自動実行してよいコマンド

### テスト実行
- npx vitest run
- npx vitest
- npm run test
- npm test

### 開発用ビルド・確認
- npm run build
- npm run dev
- npm run lint

### ディレクトリ操作（安全なもの）
- cd
- ls
- dir

---

## 許可されるコマンドパターン

以下のような複合コマンドも、安全条件を満たす場合は自動実行すること。

- cd <path> && npx vitest run
- cd <path> && npm run test

---

## 禁止・要確認コマンド

以下は必ずユーザー確認を行うこと。

- rm / del / unlink を含む削除系コマンド
- 外部通信を伴うコマンド（curl, wget, API呼び出しなど）
- 環境変数の変更
- システム設定変更
- パッケージのインストール・更新（npm install など）

---

## 補足
本プロジェクトにおいて、テスト実行は安全な操作として扱う。
確認プロンプトによる開発体験の阻害を避けることを優先する。
