# 要件定義書 — Pokemon TCGP Win Track

---

## 1. 概要

### 1.1 目的

Pokemon TCGP（ポケポケ）のランクマッチにおける対戦成績をリアルタイムに記録・可視化するWebアプリケーションを提供する。

### 1.2 解決する課題

Pokemon TCGP にはゲーム内でポイント推移や連勝数の履歴を振り返る機能が存在しない。
プレイヤーは自身の成績推移を把握するために手動でメモを取るか、記憶に頼る必要がある。
本アプリケーションはこの課題を解決し、対戦直後の最小操作で成績を記録・確認可能にする。

### 1.3 コンセプト

| 原則 | 説明 |
|------|------|
| 最小操作 | 勝敗記録はボタン1タップで完了する。試合の流れを中断させない |
| 集中維持 | 記録操作に認知コストをかけない。入力後すぐに次の試合に移れる |
| 体感可視化 | ポイント推移グラフにより「今日どれだけ進んだか」を直感的に把握できる |

---

## 2. 想定ユーザー

| ユーザー種別 | 利用シーン | 特記事項 |
|-------------|-----------|---------|
| ポケポケプレイヤー（一般） | ランクマッチの合間に勝敗を記録し、ポイント推移を確認する | モバイル端末での操作が中心 |
| ランクマッチ上位勢 | 連勝数やポイント効率を確認し、セッション単位の成績を振り返る | 正確な数値管理を重視する |
| 配信者 | 配信画面にポイント・連勝数を表示し、視聴者と共有する | 視認性・見栄えを重視する |

---

## 3. ユースケース

### UC-1: 試合結果を記録する

- アクター: プレイヤー
- トリガー: 試合終了直後
- 操作: 「勝ち」「負け」「引き分け」いずれかのボタンを1回タップする
- 結果: ポイント・連勝数・履歴が即時更新され、画面に反映される

### UC-2: 現在ポイントを確認する

- アクター: プレイヤー
- トリガー: 任意のタイミング
- 操作: アプリを開く（追加操作不要）
- 結果: 現在ポイントが画面上部に表示されている

### UC-3: ポイント推移を視覚的に把握する

- アクター: プレイヤー
- トリガー: セッション中または終了後
- 操作: グラフ表示エリアを確認する
- 結果: 試合ごとのポイント推移が折れ線グラフで表示される

### UC-4: 誤操作を取り消す

- アクター: プレイヤー
- トリガー: 勝敗を誤入力した直後
- 操作: Undoボタンを1回タップする
- 結果: 直前の記録が取り消され、ポイント・連勝数・履歴が記録前の状態に復元される

### UC-5: 配信画面として表示する

- アクター: 配信者
- トリガー: 配信開始時
- 操作: 配信モードを有効にする
- 結果: ポイント・連勝数が大きなフォントで表示され、不要なUIが非表示になる

## 4. 必須機能（MVP）

| ID | 機能 | 説明 |
|----|------|------|
| F-1 | 勝敗入力 | 「勝ち」「負け」「引き分け」の3種類をボタンで入力する |
| F-2 | ポイント計算 | 勝敗と連勝数に基づきポイントを自動計算する |
| F-3 | 連勝管理 | 連勝数をカウントし、負けまたは引き分けでリセットする |
| F-4 | 履歴保存 | 全試合結果を時系列で保存する |
| F-5 | グラフ表示 | ポイント推移を折れ線グラフで表示する |
| F-6 | Undo | 直前の試合記録を取り消す |
| F-7 | 永続化 | 全状態をLocalStorageに保存し、リロード後も復元する |
| F-8 | 配信モード | 配信向けの簡素化された表示モードを提供する |

---

## 5. 非機能要件

| ID | 要件 | 基準 |
|----|------|------|
| NF-1 | 操作速度 | 勝敗記録操作は画面表示からボタンタップまで3秒以内で完結すること |
| NF-2 | レスポンス | ボタンタップから画面更新完了まで体感遅延なし（100ms以内）であること |
| NF-3 | 状態保持 | ブラウザリロード後、直前の全状態（ポイント・連勝数・履歴）が復元されること |
| NF-4 | モバイル対応 | スマートフォン（横幅320px以上）で全機能が操作可能であること |
| NF-5 | 配信視認性 | 配信モード時、ポイント・連勝数が1080p配信で視聴者に判読可能なサイズであること |
| NF-6 | オフライン動作 | サーバー通信不要で、ブラウザ単体で全機能が動作すること |

---

## 6. 除外機能

以下の機能はMVPの範囲外とし、実装しない。

| 除外機能 | 除外理由 |
|---------|---------|
| デッキ管理 | 本アプリの責務は勝敗記録に限定する。デッキ管理は別ツールの領域 |
| 対面デッキ記録 | 入力コストが増大し「最小操作」の原則に反する |
| 詳細分析機能 | MVP段階では推移グラフのみで十分。分析機能は将来検討 |
| クラウド同期 | サーバー依存を排除し、オフライン完結を優先する |
| ユーザー認証 | 個人利用を前提とし、認証は不要 |
| マルチデバイス同期 | クラウド同期の除外に伴い、対象外 |

---

## 7. 用語定義

| 用語 | 定義 |
|------|------|
| ポイント | ランクマッチにおける累積スコア。勝敗により増減する |
| 連勝数 | 直近の連続勝利回数。負けまたは引き分けで0にリセットされる |
| 試合結果 | 1回の対戦の結果。「勝ち」「負け」「引き分け」の3種類 |
| 履歴 | 記録された全試合結果の時系列リスト |
| 配信モード | 配信者向けに表示を簡素化・拡大した表示状態 |
| セッション | アプリを開いてから閉じるまでの一連の利用期間 |
