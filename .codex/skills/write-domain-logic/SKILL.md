# Domain Logic Skill

## トリガー条件

以下の依頼時にこの skill を適用する:
- ビジネスルールの実装（勝率計算、ポイント計算、連勝判定など）
- 純粋な計算ロジックの追加・修正
- domain 層の型定義に基づくロジック実装

## TDD 前提条件

- この skill を実行する前に、対象機能のテストを先に作成済みであること。
- テスト未作成の場合は、先に write-tests skill を適用すること。

## 実装ルール

- 純粋関数として実装すること。
- 副作用を持たないこと。
- 引数と戻り値のみで処理を完結させること。
- React や外部依存を含めないこと。
- `src/domain/` 配下に配置すること。
